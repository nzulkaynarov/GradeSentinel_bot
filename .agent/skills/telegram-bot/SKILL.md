# Telegram Bot Development Skill

This document outlines the standard procedures, architectural patterns, and best practices for developing Python-based Telegram bots in our projects (such as GradeSentinel). ИИ-агенты должны использовать этот документ как руководство (System Prompt/Skill) при написании, рефакторинге или отладке кода бота.

## 1. Архитектурные паттерны (Architecture Patterns)
- **Разделение ответственности (Separation of Concerns):** Обработчики (handlers) не должны содержать сложную бизнес-логику напрямую.
- **Слои приложения:**
  - `handlers/` — маршрутизация команд, получение данных от пользователя, вызов сервисов и отправка ответа.
  - `services/` (или `core/`) — бизнес-логика (например, обработка данных Google Sheets, сложные проверки).
  - `database/` — CRUD-операции, модели SQLAlchemy/SQL, инициализация соединений.
  - `keyboards/` — генерация Inline и Reply клавиатур (кнопок).
  - `middlewares/` — перехват апдейтов до хендлеров (авторизация пользователя, прокидывание сессий БД, защита от спама).

## 2. Лучшие практики (Best Practices)
- **Асинхронность:** Все I/O операции (база данных, запросы к API) должны быть асинхронными (`async/await`), чтобы не блокировать Event Loop бота.
- **Машины состояний (FSM):** Для многошаговых диалогов (например, "Введи имя" -> "Введи номер" -> "Сохранено") обязательно используйте Finite State Machine.
- **Переменные окружения:** Все токены (`BOT_TOKEN`), ключи Google API и конфигурационные данные храните строго в `.env`. Никогда не хардкодьте секреты.
- **Обработка ошибок:**
  - Оборачивайте внешние API-вызовы в `try/except` (желательно с механизмом Exponential Backoff для сетевых ошибок).
  - Пользователь должен получать понятное сообщение об ошибке (например, "Извините, сервис временно недоступен"), а не сырой Python Traceback.
  - Логируйте все критические спады через стандартный модуль `logging`.

## 3. Технологический стек "по умолчанию"
- **Язык:** Python 3.10+ (Lightweight Slim Docker Images)
- **Фреймворк:** `aiogram` (рекомендуется для современных асинхронных ботов).
- **База данных:** SQLite + `SQLAlchemy` (Core/ORM) / `aiosqlite`.
- **Инфраструктура:** Docker + `docker-compose.yml`.

## 4. Требования к UI/UX бота
- **Тексты:** Форматируйте сообщения в HTML. Сообщения должны быть вежливыми и понятными.
- **Эмодзи:** Используйте эмодзи для расстановки акцентов, но не перегружайте ими текст (например, ✅ для успеха, ⚠️ для предупреждений).
- **Навигация:** Если у пользователя много опций, используйте древовидное меню через Inline-кнопки (с кнопками "Назад / На главную").

## 5. Инструкция для ИИ (AI Agent Instructions)
Когда тебя просят написать новый функционал для бота:
1. Сначала проверь структуру проекта: куда логичнее положить новый код (в новый файл в `handlers/` или расширить существующий).
2. Напиши логику раздельно (хендлер для связи с юзером, функцию базы данных для хранения данных).
3. Убедись, что обновлен интерфейс (клавиатуры/сообщения).
4. Проверь безопасность (имеет ли пользователь права роли Admin / Head для вызова этой команды).
