# Руководство по разработке (Development Guide)

## Требования
- **Python:** 3.10 или новее
- **База данных:** SQLite (встроена)
- **Токены:**
    - Ключ от Telegram-бота (`BOT_TOKEN` из @BotFather).
    - JSON-ключ Google Service Account для обращения к Google API.

## Подготовка к локальной разработке
1. **Клонирование репозитория:**
   ```bash
   git clone <URL>
   cd GradeSentinel_bot
   ```

2. **Настройка виртуального окружения:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Mac/Linux
   # ИЛИ для Windows: venv\Scripts\activate
   ```

3. **Установка зависимостей:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Конфигурация окружения (.env):**
   Скопируйте пример файла конфигурации:
   ```bash
   cp .env.example .env
   ```
   Укажите в `.env` необходимые ключи (BOT_TOKEN, пути к ключам БД и Google). **НИКОГДА не коммитьте `.env` в Git.**

5. **Запуск бота локально (без Docker):**
   ```bash
   python src/main.py # Либо тот файл, который является точкой входа
   ```

## Рабочий процесс и Git (Workflow)
- Ветка **`develop`**: 
  - Основная ветка разработки и песочница для тестов.
  - Рекомендуется локальное тестирование кода на Mac (или ПК) именно в этой ветке.
- Ветка **`main`**: 
  - Релизная Production ветка. 
  - Любой *Push/Merge* в эту ветку – это стабильный продукт. Код из этой ветки запускается на домашнем сервере (Raspberry Pi).

## Правила написания кода
- Оформлять документацию через Docstrings к функциям.
- По возможности следовать `PEP 8` (советуется установить линтеры: `flake8`, `black`, `isort`).
- Перед коммитом проверять работоспособность локально.
