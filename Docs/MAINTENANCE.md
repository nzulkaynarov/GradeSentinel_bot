# Обслуживание и Деплой (Deployment & Maintenance)

## Инфраструктура
Проект (Bot + DB + Monitor) упакован в **Docker Image**. 
- Основной целевой сервер: Raspberry Pi 3B (Home Server).
- Базовый образ: `python:3.10-slim` (оптимизированный и легковесный).

## Запуск в Production
1. Перенесите файлы проекта (или сделайте `git pull` ветки `main`) на Raspberry Pi.
2. Убедитесь, что в корне проекта находятся:
   - Файл `.env` с переменными окружения.
   - Файл с ключами `google_credentials.json` (или укажите правильный путь в конфиге).
3. Соберите и запустите контейнер в фоне:
   ```bash
   docker-compose up -d --build
   ```
   *Параметр `restart: always` в конфигурации обеспечит автоматический перезапуск бота при перезагрузке малины или падении контейнера.*

## Администрирование через Docker

- **Просмотр логов в реальном времени:**
  ```bash
  docker-compose logs -f
  # Для последних 100 строк можно использовать:
  docker-compose logs --tail 100 -f
  ```

- **Остановка работы (Shutdown):**
  ```bash
  docker-compose down
  ```

- **Мягкий перезапуск проекта (Рестарт сервисов):**
  ```bash
  docker-compose restart
  ```

## Резервное копирование (Backups)
Главная ценность — база данных `database.db` SQLIte, хранящая учеников и пользователей. Папка с БД прокидывается (mount) из контейнера на сервер для сохранности данных при пересборке.

Для создания бэкапа достаточно просто скопировать файл БД:
```bash
cp /path/to/project/data/database.db /path/to/backups/db_backup_$(date +%F).db
```
*(Рекомендуется настроить Cron на сервере для автоматического снятия дампа каждую ночь)*.
