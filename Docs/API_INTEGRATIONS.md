# Интеграции с API (API Integrations)

## 1. Telegram Bot API
Бот взаимодействует с платформой Telegram для приема сообщений и отправки уведомлений:
- Для получения обновлений (Updates) может использоваться **Polling** (удобно для локальной машины и запуска на Raspberry Pi за NAT). 
- Все запросы должны быть асинхронными, чтобы обеспечивать высокую пропускную способность.

## 2. Google Sheets API v4
Фундамент модуля **Antigravity Monitor**. Бот читает ячейки для выявления новых оценок.

### Подготовка и аутентификация (Service Account)
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект.
3. В библиотеке API включите **Google Sheets API** и **Google Drive API** (опционально, требуется для расширенного доступа к файлам).
4. Создайте Service Account (Учетную запись службы) в разделе *IAM & Admin*.
5. Сгенерируйте и скачайте JSON-ключ. Поместите его в проект (часто называется `credentials.json`).
6. **КРИТИЧЕСКИ ВАЖНО:** Чтобы бот смог читать вашу Google Таблицу через Service Account, вы должны нажать "Поделиться" в интерфейсе этой таблицы и выдать права Читателя/Редактора email-адресу Service Account (формата `*@*.iam.gserviceaccount.com`).

### Rate Limits и Exponential Backoff
Google Sheets API имеет строгие лимиты (квоты) на чтение/запись данных в минуту.
- В двигатель интегрирован **Exponential Backoff**. 
- При получении ошибки `HTTP 429 Too Many Requests` или `HTTP 500/503` монитор автоматически ставит выполнение на паузу, постепенно увеличивая задержку (например: 1с, 2с, 4с, 8с), чтобы API успело "остыть". Это спасает систему от вечной блокировки.

### Валидатор ссылок Google Sheets
В боте реализован механизм валидации. Прежде чем сохранить ссылку на профиль/семью, бот:
1. Парсит URL с помощью регулярных выражений, извлекая `spreadsheetId`.
2. Пытается тестово получить метаданные таблицы.
3. Если доступ есть, ссылка сохраняется. Если выдается ошибка доступа (HTTP 403 / 404), бот сообщает пользователю, что нужно выдать доступ для Service Account.
