# Архитектура проекта GradeSentinel (Antigravity)

## Концепция
GradeSentinel — это Telegram-бот для автоматического сбора и рассылки оценок или статусов из Google Таблиц. Бот использует локальную СУБД SQLite для хранения пользователей и их связей, а также делает асинхронные запросы к Google Sheets (Monitor) для отслеживания изменений.

## Структура Базы Данных (SQLite + SQLAlchemy)
База данных построена по реляционной модели (структура Многие-ко-Многим). Основные сущности:
1. **Users / Parents** - пользователи Telegram, взаимодействующие с ботом.
2. **Students** - ученики/студенты, чьи оценки отслеживаются.
3. **Families** - группы, связывающие пользователей (родителей) и учеников.
4. **Family Links** - таблицы связей для обеспечения ролевой модели.

### Мульти-Ролевая Архитектура (Умный доступ)
- **Super Admin** - имеет глобальный доступ ко всем системам бота (управление семьями и административными функциями).
- **Head of Family** (`head_id` как атрибут объекта Family) - управляет составом конкретной семьи, может добавлять и удалять детей участников.
- **Senior / Parent** - участник семьи, который может получать уведомления и просматривать информацию, но не может менять состав семьи.

*Один человек может управлять несколькими семьями или состоять в разных семьях одновременно.*

## Главные Компоненты Системы

### 1. Telegram Bot (Центр Управления)
   - Принимает команды и сообщения через Telegram Bot API.
   - **`auth_handler`**: система верификации пользователей (например, по номеру телефона) перед началом работы.
   - **Умный конструктор кнопок UI**: динамическое построение меню, которое объединяет административные, родительские и семейные функции в единый интерфейс.

### 2. Модуль Monitor (Двигатель Antigravity)
   - **Snapshot-сравнение**: Модуль периодически делает срез (снимок) данных из Google Таблицы и сравнивает его с предыдущим состоянием, находя изменения в ячейках.
   - **Защита от потери**: Сохранение истории отчетов для гарантии доставки даже при временных сбоях.
   - **Надежность сети**: Обработка ошибок Google API (Rate Limits, Network errors) с использованием умного ожидания (Exponential Backoff).
   - **Уведомления**: Гибкая система отправки красивых уведомлений (с использованием HTML-разметки и эмодзи).
