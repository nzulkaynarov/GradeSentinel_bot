# 📄 Техническое Задание (Terms of Reference): GradeSentinel v2.0

> **GradeSentinel** — это автономная, масштабируемая система мониторинга школьной успеваемости. Проект выступает надежным "мостом" между Google Таблицами (где выставляются оценки) и мессенджером Telegram (интерфейс родителей и администраторов), обеспечивая мгновенную доставку уведомлений с использованием строгой ролевой модели доступа.

---

## 1. Цели и задачи продукта

*   **Автоматизация рутины:** Полное исключение необходимости ручной проверки электронных дневников родителями.
*   **Оперативность (Real-time monitoring):** Гарантированная доставка уведомлений об изменениях в оценках в течение 5 минут.
*   **Безопасность и Приватность:** Доступ к персональным данным детей предоставляется исключительно после [верификации](#42-контур-родителя-client-side) пользователя по номеру телефона (Contact Sharing).
*   **Масштабируемость:** Архитектура, позволяющая обслуживать десятки семей в рамках одного Telegram-бота без взаимной блокировки процессов (Non-blocking I/O).

---

## 2. Архитектура системы и Стек технологий

Проект спроектирован с упором на переносимость и отказоустойчивость. Вся система упакована в **Docker-контейнер**, что позволяет развернуть её как на домашнем сервере (Raspberry Pi), так и на облачных VPS.

| Компонент | Технология / Инструмент | Описание |
| :--- | :--- | :--- |
| **Язык разработки** | Python 3.10+ | Основной язык (асинхронный подход). |
| **База Данных** | SQLite3 + SQLAlchemy | Легковесная реляционная СУБД (хранится в Docker Volume для персистентности). |
| **Telegram API** | `aiogram` / `telebot` | Асинхронное взаимодействие с Telegram Bot API. |
| **Google API** | Google Sheets API v4 | Чтение данных таблиц через сервисный аккаунт (Service Account) с реализацией **Exponential Backoff**. |
| **Инфраструктура** | Docker + Docker Compose | Изолированная среда выполнения (`python:3.10-slim`). |

> 📖 **Связанная документация:** Подробное описание развертывания см. в [Docs/MAINTENANCE.md](Docs/MAINTENANCE.md) и [Docs/DEVELOPMENT.md](Docs/DEVELOPMENT.md).

## 3. Модель данных (Database Schema)

Система базируется на реляционной модели с поддержкой связей **"Многие-ко-многим" (Many-to-Many)**, что обеспечивает гибкое управление [Семьями].

### Основные сущности:
1.  **`Families` (Семьи):** Группирующая корневая сущность. Хранит данные о подписке (`subscription_end`) и глобального Главу семьи (`head_id`).
2.  **`Parents` (Родители):** Пользователи Telegram. 
    *   Атрибуты: `id`, `fio`, `phone` *(unique)*, `telegram_id` *(unique)*, `role` (`admin` или `senior`).
3.  **`Students` (Ученики):** Профили детей, за которыми ведется наблюдение.
    *   Атрибуты: `id`, `fio`, `spreadsheet_id` (ID Google-файла).
4.  **`Family_Links`:** Связующая таблица для реализации связи M:N (привязка `Parents` и `Students` к конкретной `Family`).
5.  **`Snapshots` (grade_history):** Техническая таблица. Хранит последнее известное состояние ячеек оценок из Google Таблиц для Snapshot-сравнения.

> 📖 **Связанная документация:** Глубокий разбор структуры БД доступен в [Docs/ARCHITECTURE.md](Docs/ARCHITECTURE.md).

## 4. Пользовательский путь (User Flow) и Ролевая модель

В системе реализован "Умный доступ" (Smart Access), разделяющий интерфейс и права в зависимости от роли пользователя.

### 4.1. Административный контур (Admin / Super Admin)
*   **Создание Семьи:** Супер-администратор (владелец бота) через встроенное меню инициирует создание новой сущности `Family`.
*   **Делегирование:** Сразу при создании семьи, Супер-админ назначает Родителя (со статусом Главы семьи — _Head_).
*   **Рассылка (Broadcast):** Возможность отправки массовых системных анонсов всем зарегистрированным пользователям.
*   **Поддержка (Support):** Двусторонняя связь. Сообщения пользователей пересылаются в закрытую админ-группу. Ответ (через 'Reply' в группе) доставляется пользователю от имени бота. 

### 4.2. Контур Родителя (Client Side)
1.  **Авторизация:** Команда `/start` вызывает принудительный запрос контакта (кнопка *«Отправить номер»*).
2.  **Верификация (Security Check):** 
    *   Бот ищет номер телефона в таблице `Parents`.
    *   _Не найден:_ Выдача сообщения об отказе в обслуживании.
    *   _Найден:_ Регистрация `telegram_id` в базе и открытие доступа к основному меню.
3.  **Управление Семьей (только для роли Head):** Глава семьи имеет право автономно добавлять в свою `Family` новых родственников (бабушек/дедушек) и детей.
4.  **Мониторинг:** Бот начинает цикл фоновой проверки дневников для привязанных детей.

## 5. Модуль мониторинга (Antigravity Engine)

Двигатель системы работает в фоне, используя механизм Polling.

*   **Алгоритм Snapshot-сравнения:** Каждые 5 минут скрипт скачивает данные из рабочей вкладки ученика (например, "Сегодня") и сравнивает хэш/содержимое ячеек с последним снимком, сохраненным в `Snapshots` БД.
*   **Точечные Уведомления:** При обнаружении новой записи бот немедленно формирует и отправляет сообщение всем родителям, привязанным к семье данного ученика:

    ```text
    🔔 Новая оценка!
    👨‍🎓 Ученик: Заур
    📚 Предмет: Физика
    📝 Тип: Контрольная работа
    ⭐ Оценка: 5
    ```

> 📖 **Связанная документация:** Подробности работы с квотами Google (Exponential Backoff) описаны в [Docs/API_INTEGRATIONS.md](Docs/API_INTEGRATIONS.md).

## 6. Дерево директорий (Filesystem Structure)

Кодовая база разделена на логические слои (Separation of Concerns).

```text
/GradeSentinel
├── Dockerfile                  # Сборка образа (Production/Dev)
├── docker-compose.yml          # Оркестрация сервисов и Volume
├── requirements.txt            # Зависимости Python
├── .env                        # Секретные ключи (ИСКЛЮЧЕН ИЗ GIT)
├── Docs/                       # 📚 Документация (Схемы, Руководства)
├── config/
│   └── credentials.json        # Ключ авторизации Google Service Account
├── data/
│   └── sentinel.db             # База данных SQLite (пробрасывается в Volume)
└── src/                        # 💻 Исходный код приложения
    ├── main.py                 # Точка входа (Entry point)
    ├── database/               # SQL-алхимия, модели (ORM), миграции
    ├── services/               # Бизнес-логика, интеграция с Google Sheets
    ├── handlers/               # Маршрутизация Telegram-команд (роутеры)
    ├── keyboards/              # Генераторы UI (Inline-/Reply-кнопки)
    └── monitor/                # Движок фонового мониторинга (Antigravity)
```

## 7. Вектор развития (Roadmap & Commercials)

*   **SaaS-модель:** Полностью автономное добавление семей через Telegram-интерфейс (без участия Super Admin и остановки бота).
*   **Биллинг:** Внедрение демо-периодов (использование поля `subscription_end` БД) и механизмов автоматической блокировки/напоминаний об оплате.
*   **Web Dashboard:** Разработка дополнительного UI на Flask/FastAPI для визуализации успеваемости (графишки, тренды) с доступом по Telegram WebApp (Mini Apps).
*   **AI-Аналитика:** Интеграция с LLM (OpenAI) для текстового анализа успеваемости (_«Внимание: Тренд на снижение оценок по алгебре за последние 2 недели»_).
