# 📝 Дизайн-документ: GradeSentinel v2.0 Мониторинг и Обработка Данных

Обзор: Этот документ описывает детали реализации для системы авто-мониторинга оценок (Antigravity), включая обработку "грязных" данных из Google Sheets.

## 1. Проблема: "Грязные" данные
Преподаватели могут вводить не только цифры, но и текст (например, "5-", "н", "болел", "5="). Это ломает статистику и парсер.

### Решение: Гибридный подход
1. **Чистка (Sanitization):** Специальный модуль `data_cleaner.py` будет фильтровать все входящие символы из ячеек. Если оценка "5-" или "5=" — она сохраняется в базу для статистики как строгое число `5.0`.
2. **Оригинальные данные:** Если из ячейки невозможно достать числовую оценку (например, "болел" или "н"), то бот присылает оригинальный текст в виде примечания, не сохраняя при этом `grade_value` для статистики.

## 2. Архитектура Данных: Таблица `grade_history`
*   `id` (INTEGER PRIMARY KEY)
*   `student_id` (INTEGER)
*   `subject` (TEXT)
*   `grade_value` (REAL, NULL) - очищенная цифра для графиков.
*   `raw_text` (TEXT) - оригинальное значение из ячейки.
*   `cell_reference` (TEXT) - координаты ячейки (например, "С5"). *Используется для предотвращения дублирования уведомлений.*
*   `date_added` (TIMESTAMP)

## 3. Мониторинг (Polling Mechanism)
*   **Частота:** Каждые 5 минут.
*   **Диапазон:** Бот будет сканировать только вкладку **"Сегодня"** для выявления новых оценок (для экономии лимитов и ускорения работы). 
*   **По запросу:** На вкладку "Все оценки" или другие вкладки бот будет обращаться только по прямому запросу пользователя (через меню в Telegram).
*   **Алгоритм обнаружения новых оценок:** Для каждой непустой ячейки проверяется: есть ли в базе запись с данным `student_id` и `cell_reference`. Если нет — отправляется уведомление.
